<objective>
Close two gaps identified by the v1.0 milestone audit:

1. **SV-09**: `SvCtaSection` on `/safe-verify` is a custom 25-line standalone section with only a `/contact` link — no calculator CTA. Replace it with a thin 4-line wrapper over `PageCtaSection` (identical to `DscCtaSection`), which provides both "See the value for your portfolio" (→ `/portal/calculator`) and "Request a demo" (→ `/#demo`).

2. **PORT-04**: `PortalLoginTracker` is only mounted in `portal/page.tsx` (the dashboard). The primary auth funnel (homepage teaser → `/portal/login?callbackUrl=/portal/calculator`) redirects post-auth directly to `/portal/calculator`, bypassing the dashboard entirely. Moving `PortalLoginTracker` to `src/app/(portal)/layout.tsx` ensures it fires on every authenticated portal page load regardless of callbackUrl destination.

Purpose: Both gaps prevent conversion signal from reaching the sales team. SV-09 removes the calculator entry point from a product page; PORT-04 means Attio never sees portal_login events for the primary funnel path.

Output: Three modified files — SvCtaSection.tsx (replaced), layout.tsx (tracker added), portal/page.tsx (tracker removed).
</objective>

```typescript
'use client'

import { useEffect } from 'react'
import { firePortalLogin } from '@/app/actions/attio'
import { createClient } from '@/lib/supabase/client'

export function PortalLoginTracker() {
  useEffect(() => {
    createClient().auth.getUser().then(({ data }) => {
      if (data.user?.email) {
        firePortalLogin(data.user.email)
      }
    })
  }, [])
  return null
}
```
