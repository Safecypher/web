## Gaps (Critical)

### GAP 1: SV-09 — Safe Verify CTA missing calculator and demo links

**Requirement:** SV-09 — CTAs: calculator link + demo request
**Phase:** 04 — Safe Verify

`src/components/marketing/safe-verify/SvCtaSection.tsx` was modified by commit `6f8fb94` after Phase 04-04 completed. The original implementation was a thin wrapper over `PageCtaSection` (as per 04-04-SUMMARY). The current file is a custom 25-line section with "Start with Safe Verify" messaging that links only to `/contact`.

**Impact:** `/safe-verify` page has no CTA linking to the ROI calculator (`/portal/calculator`) or the homepage demo form (`/#demo`). All other product pages (platform, DSC) provide both.

**Fix:** Replace `SvCtaSection.tsx` with a thin wrapper over `PageCtaSection`, or update the custom component to include both CTAs.

---

### GAP 2: PORT-04 — portal_login event skipped in primary funnel

**Requirement:** PORT-04 — Sales team Attio notification on new portal signup
**Phase:** 07 — Add value calculators to the portal

`PortalLoginTracker` is mounted only in `src/app/(portal)/portal/page.tsx`. The primary auth funnel (homepage teaser → `/portal/login?callbackUrl=/portal/calculator?portfolioSize=...`) redirects directly to `/portal/calculator` after authentication, bypassing `/portal/page.tsx`. The `portal_login` Attio event never fires for this main path.

**Impact:** Sales team does not receive Attio notification when a prospect authenticates via the homepage teaser (the primary acquisition funnel). Events only fire when users manually navigate to the portal dashboard.

**Fix:** Move `PortalLoginTracker` to the portal layout or the Supabase auth callback route, so it fires regardless of `callbackUrl` destination.
