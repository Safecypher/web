  <action>
Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  pull_request:
    branches:
      - main
      - develop

jobs:
  ci:
    name: Lint, Type-check, Build
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type-check
        run: npx tsc --noEmit

      - name: Build
        run: npm run build
```

CRITICAL notes:
- `npm run lint` MUST be an explicit step â€” do NOT rely on `next build` to run ESLint. In Next.js 16, `next build` does NOT run ESLint automatically.
- `npx tsc --noEmit` uses the project's local TypeScript version (installed by create-next-app). Do not add a separate `typescript` dev dependency â€” it's already in package.json.
- `npm ci` is used (not `npm install`) for reproducible installs in CI.
- `cache: npm` caches node_modules between runs via the package-lock.json hash.
- Node 22 matches the LTS version appropriate for Next.js 16.
- The workflow triggers on PRs to both `main` and `develop` branches. Branch protection rules in GitHub will block merges if this check fails â€” that is configured in the GitHub repository settings, not here.

Verify the `npm run lint` script exists in package.json (create-next-app should have added it as `"lint": "next lint"`). If missing, add it.
