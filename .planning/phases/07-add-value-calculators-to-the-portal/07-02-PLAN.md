    Create src/lib/calculator/engine.ts — pure function implementing all 10 formula steps from RESEARCH.md "Spreadsheet Formula Engine" section.

    Use the complete implementation provided in RESEARCH.md "Calculator Engine — Core Implementation Pattern" code example verbatim. Key implementation points:

    Step 1: cvvRequired_D = debitCNPTransactions * debitCvvPct (not accounts — transactions)
    Step 2: fraudCases_D = cvvRequired_D * debitFraudRate
    Step 3: issuerLossPerCase_D = debitLossPerCase * issuerPct; instCostPerCase_D uses institutionalCostMethod switch
    Step 4: totalFraudCost_D = (fraudCases_D * issuerLossPerCase_D) + (fraudCases_D * instCostPerCase_D)

    Step 6: Ongoing uses ongoingAdoptionRate — same fee formula with ongoingAdoptionRate
    Step 7: Interchange uses embedded constants 0.0005/0.21 for debit and 0.0234/0.10 for credit
    Step 8: avgYr1HaloFactor = monthsToPeak >= 12 ? 1 + (peakHaloMultiplier * 0.5) : 1 + (peakHaloMultiplier * ((monthsToPeak/2)/12))
    Step 9: totalYr1Savings = netSavingsYr1 + yr1HaloBonus_D + yr1HaloBonus_C  [NO interchange]
    Step 10: sensitivityRows formula: (totalFraudCost_combined * rate) - ((cvvRequired_D + cvvRequired_C) * rate * feePerTx)
